01. About Me

I am a Staff Engineer at Samsung Research, working on LLM post-training (SFT/RLHF), on-device health LLMs, and vision-language-action & world models for robotics. 04. Things I’ve Built

  • Explainable ILI Forecasting with Self-Instructed LLMs

    First-author research project: combines ILI time series with CDC FluView / Japan IDWR surveillance reports, using GPT-generated explanation labels to LoRA fine-tune LLaMA-3.1-70B (LLaMA-Factory, DeepSpeed ZeRO-3) so the model produces predictions together with the reasoning behind them. Includes an LLM-judge evaluation of explanation quality and classic TSF baselines (DLinear, PatchTST, etc.). Submitted to IEEE BIBM 2026.

    • LLaMA-3.1-70B
    • LoRA
    • Self-Instruct
    • LLaMA-Factory
    • DeepSpeed
    • vLLM

  • csat-eval: Korean CSAT Benchmark for LLMs

    An evaluation toolkit that grades LLMs on Korean college entrance exam (CSAT) math/English/Korean problems across four backends (OpenAI API, HuggingFace, vLLM, gpt-oss), with PDF-to-LaTeX dataset building and robust answer extraction. Also includes an SFT data pipeline that generates math solutions at scale and keeps only automatically verified ones (rejection sampling).

    • LLM Evaluation
    • vLLM
    • Rejection Sampling
    • SFT Data Generation
    • Python

  • Hierarchical AI Agent Framework

    A hierarchical multi-agent chatbot framework built for an Incheon International Airport chatbot project: a main agent recursively delegates tasks to sub-agents (browser, PDF, vector-DB), and every action is emitted as executable Python code (CodeAct/ReAct paradigm). Native functions, MCP server tools, and sub-agents are all exposed to the LLM as uniform callable tools.

    • Multi-Agent
    • CodeAct/ReAct
    • MCP
    • LangChain
    • Python

  • slime + NeMo Gym RL Integration

    Design and code-flow analysis for integrating THUDM’s slime (Ray + SGLang rollout + Megatron PPO training) with NVIDIA NeMo Gym (environments, reward verifiers, multi-turn tool orchestration) into a single RLHF/RLVR post-training pipeline. Documents the full rollout-train-weight-sync cycle and the adapter layer connecting slime’s custom reward/generation hooks to NeMo Gym servers.

    • RLHF/RLVR
    • PPO
    • Ray
    • SGLang
    • Megatron
    • NeMo Gym

  • Health LLM Evaluation Harness

    Built the model-selection evaluation for an on-device health assistant LLM at Samsung Research: streaming latency profiling (TTFT, tokens/sec, thinking-vs-content token efficiency) plus automated quality scoring via LLM-as-a-judge with rubric-based metrics. Used to compare candidate internal and open-weight models on representative health-assistant scenarios.

    • LLM-as-a-Judge
    • vLLM
    • Latency Profiling
    • asyncio
    • Evaluation Design

  • Self-Instruct Fine-Tuning for Korean Term Extraction

    Delivered a Korean terminology-extraction LLM for an enterprise client: synthesized ~10k (text → term list) training pairs with GPT-4 self-instruct, fine-tuned SOLAR-10.7B with 8-bit quantization + LoRA, then merged the adapter for deployment. Handled tokenizer vocabulary extension and loss masking for instruction-style training.

    • SOLAR-10.7B
    • LoRA
    • 8-bit Quantization
    • Self-Instruct
    • HuggingFace

  • On-Premise LLM Routing for Enterprise Claude Code

    Enabled the full Claude Code agent experience inside a security-restricted enterprise network by redirecting all model calls — without modifying the official client — to an internally hosted open-weight LLM. Built the serving stack with SGLang (tensor parallelism, speculative decoding) behind a LiteLLM proxy that translates between Anthropic and OpenAI API formats, plus a warm-up daemon to prevent cold starts.

    • SGLang
    • LiteLLM
    • Speculative Decoding
    • LLM Serving
    • Node.js/Python
